预训练的核心任务，是根据前文预测下一个 token。
模型一开始并不知道世界知识，它只会看到大量文本，然后不断调整参数。
每个 batch 都是从长文本里切出来的一段连续窗口。
输入是一段序列，目标是右移一位后的同一段序列。
如果模型把下一个 token 预测得更准，损失就会下降。
如果模型预测错误，梯度下降会推动参数往更好的方向更新。

即使是一个很小的模型，也能完整演示预训练流程。
分词会把原始文本变成 token id。
嵌入层会把 token id 变成向量。
注意力机制让当前位置读取前文中的有用信息。
因果掩码会阻止模型偷看未来位置。
输出层会把隐藏状态映射成整个词表上的 logits。
交叉熵损失会把 logits 和正确答案进行比较。

训练时，模型会反复执行同样的步骤。
读取一个 batch。
执行前向传播。
计算损失。
执行反向传播。
更新参数。
继续下一步。

在训练刚开始时，模型生成的文本通常很混乱。
随着训练步数增加，一些局部结构会慢慢出现。
模型会逐渐学会空格、标点、重复词语，以及简单的语序模式。
在非常小的语料上，模型通常会过拟合。
但对于教学用途的 MVP 来说，这反而有助于你看到模型确实学到了训练文本里的模式。

这个项目的目标不是做一个强大的模型，而是理解预训练到底在做什么。
真实的大模型预训练会使用更大的语料、更好的 tokenizer、更多层、更大的隐藏维度，以及分布式训练。
但是最核心的目标并没有变化，依然是 next token prediction。
模型看到一段上下文，然后预测下一个 token。
这个目标看起来简单，却支撑了大语言模型的大部分能力。

下面这段内容会重复表达相似意思，为小模型提供更多模式信号。
预训练本质上是一种自监督学习。
它不需要人工额外标注答案。
因为文本里的下一个 token 本身就是监督信号。
优化器会一步一步改进模型参数。
验证损失可以帮助我们判断模型是否只是在死记训练样本。
文本生成则是一种直观的观察方式，可以看到模型到底学会了什么。

再换一种说法继续强化模式。
预训练是在构建一个对语料分布的压缩统计模型。
模型不会显式保存规则清单。
它保存的是一组参数，这些参数会对下一个 token 产生更合理的 logits。
logits 越合理，损失通常越低。
损失越低，生成结果通常越连贯。
一个很小的模型不会突然变聪明，但它会逐渐从随机输出变成带有局部规律的输出。

再补充一个练习段落。
上下文长度很重要，因为模型只能看到窗口范围内的 token。
模型规模也很重要，因为更多层和更大的维度意味着更强的表示能力。
优化参数同样重要，因为学习率和 batch size 会影响训练稳定性。
数据质量也很重要，因为模型只能学习语料中真正出现过的模式。
评估同样重要，因为只看训练损失很容易忽略过拟合。

最后再提醒一次。
这份语料是故意做小的。
等你理解了整个流程之后，可以替换成你自己的文本。
第一步先把链路跑通。
之后再逐步升级 tokenizer、dataset、model、trainer 和采样策略。
